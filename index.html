Configure Authentication

ⓘ All endpoints require a Bearer token. Get your key from cloud.blackbox.ai → Profile → BLACKBOX API Token. Keys start with bb_.

Set up a Postman Environment

  • 1
    Create a new Environment

    In Postman: Environments sidebar → + → name it Blackbox AI.

  • 2
    Add the variables below

    Set the Current Value for each variable. Secret values can be marked as "secret" so Postman masks them.

  • 3
    Set Collection-level auth

    Open the imported collection → Authorization tab → Type: Bearer Token → Token: {{apiKey}}. All child requests inherit this.

Variable Example / Default Required Description
apiKey bb_xxxxxxxxxxxxxx REQUIRED Your Blackbox API key from cloud.blackbox.ai
baseUrl https://cloud.blackbox.ai/api optional API base URL (already set in the spec)
taskId 9qQe2F8Z_nXx9-eJA0BD6 optional Re-usable task ID for GET / PATCH / stream calls

Available Endpoints

All 7 endpoints are imported automatically. Here is a quick reference:

POST /tasks

Create Task

Launch a new agent task against a GitHub repo. Supports single-agent and multi-agent (2–5) parallel modes.

GET /tasks

List Tasks

Paginated list with filtering by type (all, tasks, batch) and status (active, completed, failed, stopped).

GET /tasks/{taskId}

Get Task Details

Full task object including logs, diff stats, multi-agent executions, PR number, and branch info.

PATCH /tasks/{taskId}

Cancel Task

Stop a running task. Only works when status is processing or saving.

GET /tasks/{taskId}/status

Get Task Status

Lightweight polling endpoint returning status, progress, and timestamps. Use this instead of getTask for frequent polling.

POST /tasks/{taskId}/continue

Continue Task

Send a follow-up prompt to an existing task. The agent resumes with full prior context and history.

GET /tasks/{taskId}/stream

Stream Task Logs

Real-time SSE log stream. Set Accept: text/event-stream in Postman headers. Supported since Postman v10.18.

Example Requests

POST /tasks — Single Agent 
// POST https://cloud.blackbox.ai/api/tasks
// Authorization: Bearer {{apiKey}}
// Content-Type: application/json

{
  "prompt": "Add a dark mode toggle to the React app",
  "repoUrl": "https://github.com/my-org/my-repo.git",
  "selectedBranch": "main",
  "selectedAgent": "blackbox",
  "selectedModel": "blackbox-pro",
  "installDependencies": false,
  "maxDuration": 300
}
POST /tasks — Multi-Agent Parallel 
// POST https://cloud.blackbox.ai/api/tasks
// Authorization: Bearer {{apiKey}}
// Content-Type: application/json

{
  "prompt": "Refactor auth module to use JWT middleware",
  "repoUrl": "https://github.com/my-org/my-repo.git",
  "selectedBranch": "main",
  "selectedAgents": [
    {
      "agent": "claude",
      "model": "blackboxai/anthropic/claude-sonnet-4.5"
    },
    {
      "agent": "blackbox",
      "model": "blackboxai/blackbox-pro"
    }
  ]
}
GET /tasks/{taskId}/status — Poll 
// GET https://cloud.blackbox.ai/api/tasks/{{taskId}}/status
// Authorization: Bearer {{apiKey}}
// No request body needed

// Example 200 response:
{
  "taskId": "9qQe2F8Z_nXx9-eJA0BD6",
  "status": "processing",
  "progress": 45,
  "inProgress": true,
  "isDone": false,
  "error": null
}
GET /tasks/{taskId}/stream — SSE 
// GET https://cloud.blackbox.ai/api/tasks/{{taskId}}/stream
// Authorization: Bearer {{apiKey}}
// Accept: text/event-stream
// Query params: ?fromIndex=0&includeStatus=true

// SSE events streamed in real-time:
event: connected
data: {"taskId":"9qQe2F8Z_nXx9-eJA0BD6","fromIndex":0}

event: log
data: {"index":0,"log":{"message":"Cloning repository...","step":"git_cloning"}}

event: status
data: {"status":"processing","error":null}

event: complete
data: {"status":"completed","totalLogs":24}
⚠ SSE streaming requires Postman v10.18+. For older versions, use cURL with the -N flag (no buffering): curl -N -H "Authorization: Bearer $KEY" https://cloud.blackbox.ai/api/tasks/ID/stream
POST /tasks/{taskId}/continue — Follow-up 
// POST https://cloud.blackbox.ai/api/tasks/{{taskId}}/continue
// Authorization: Bearer {{apiKey}}
// Content-Type: application/json

{
  "currentPrompt": "Also add unit tests for the new auth module",
  "selectedAgent": "claude",
  "selectedModel": "blackboxai/anthropic/claude-sonnet-4.5",
  "maxDuration": 300
}

Postman Tips & Power Workflow

Auto-extract taskId in Tests tab

In the Tests tab of the Create Task request, add this script so Postman auto-saves the task ID for subsequent requests:

const r = pm.response.json();
pm.environment.set("taskId", r.task.id);
console.log("taskId set:", r.task.id);

Poll until complete with Collection Runner

Use Postman Flows or a Collection Runner loop on the status endpoint. In the Tests tab add:

const r = pm.response.json();
if (!r.isDone) {
  // keep polling
  postman.setNextRequest("Get Task Status");
}

Error Codes to Know

401 — Invalid or missing API key
402 — Insufficient credits
400 — Invalid agent or bad request body
404 — Task not found
502 — GitHub API error / upstream failure
